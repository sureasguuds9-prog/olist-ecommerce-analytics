# Dashboard в Yandex DataLens

Для проекта подготовлены отдельные CSV-витрины, которые можно загрузить в Yandex DataLens без объединения исходных таблиц Olist.

## Подготовленные витрины

| Файл | Назначение |
|---|---|
| `data/executive_kpis.csv` | основные показатели проекта |
| `data/monthly_sales.csv` | динамика заказов, клиентов, GMV и среднего чека |
| `data/category_performance.csv` | показатели товарных категорий |
| `data/delivery_quality.csv` | качество доставки и отзывы по штатам |
| `data/cohort_retention.csv` | когортный retention в длинном формате |
| `data/rfm_segments.csv` | агрегированные RFM-сегменты |

Повторно создать витрины из исходных данных:

```bash
python src/prepare_datalens_data.py
```

## Рекомендуемая структура дашборда

### Страница 1. Главное

Карточки:

- доставленные заказы;
- уникальные клиенты;
- GMV;
- средний чек;
- доля повторных клиентов;
- доля опозданий.

Графики:

- GMV по месяцам;
- количество заказов по месяцам;
- топ-10 категорий по GMV.

Источник: `executive_kpis.csv`, `monthly_sales.csv`, `category_performance.csv`.

### Страница 2. Продажи

Графики:

- GMV и количество заказов по месяцам;
- средний чек по месяцам;
- категории по GMV;
- категории по числу заказов;
- средняя оценка по категориям.

Фильтры:

- месяц покупки;
- товарная категория.

Источник: `monthly_sales.csv`, `category_performance.csv`.

### Страница 3. Retention и клиенты

Графики:

- тепловая карта когортного retention;
- количество клиентов по RFM-сегментам;
- средняя ценность по RFM-сегментам;
- средняя давность последней покупки.

Фильтры:

- когорта первой покупки;
- RFM-сегмент.

Источник: `cohort_retention.csv`, `rfm_segments.csv`.

### Страница 4. Доставка и отзывы

Графики:

- доля плохих отзывов: вовремя и с опозданием;
- средняя оценка: вовремя и с опозданием;
- доля опозданий по штатам;
- среднее время доставки по штатам.

Фильтры:

- штат клиента;
- статус доставки.

Источник: `delivery_quality.csv`.

## Как собрать в DataLens

1. Откройте [Yandex DataLens](https://datalens.yandex.ru/).
2. Создайте новое подключение типа **Файлы**.
3. Загрузите CSV из папки `datalens/data/`.
4. Для каждого файла создайте отдельный датасет.
5. Проверьте типы полей:
   - даты — `Дата`;
   - GMV и средний чек — `Дробное число`;
   - доли и retention — `Дробное число` с отображением в процентах;
   - заказы и клиенты — `Целое число`.
6. Создайте чарты по макету выше.
7. Соберите чарты в один дашборд с четырьмя вкладками.
8. Опубликуйте дашборд и добавьте публичную ссылку в основной `README.md`.

## Рекомендации по оформлению

- Используйте один основной цвет для продаж и отдельный красный акцент для опозданий и плохих отзывов.
- На странице «Главное» оставьте не более 6 KPI-карточек.
- Добавляйте краткий бизнес-вывод рядом с каждым ключевым графиком.
- Не перегружайте страницы таблицами: подробности лучше показывать через фильтры и всплывающие подсказки.
